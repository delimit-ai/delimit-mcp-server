# ============================================================================
# bundle-allowlist.txt  —  THE SECURITY BOUNDARY OF THE delimit-cli npm PACKAGE
# ============================================================================
# This file IS the security boundary. Adding a path here SHIPS it to the PUBLIC
# npm package (delimit-cli) AND to the PUBLIC git repo (delimit-ai/delimit-mcp-
# server). Do NOT add a path unless it is (a) required by the public CLI / MCP
# server surface, (b) non-proprietary, and (c) not another venture's code.
#
# FAIL-CLOSED MODEL (replaces the old fail-OPEN `gateway/` + blocklist):
#   * package.json "files" ships ONLY the paths listed here.
#   * scripts/sync-gateway.sh copies ONLY these paths into the working tree.
#   * scripts/check-bundle-classification.sh FAILS if any gateway/ai source file
#     is in NEITHER this file NOR bundle-internal-exclude.txt (unclassified).
#   * scripts/check-bundle-parity.sh FAILS if `npm pack` ships anything not
#     exactly equal to this allowlist.
#
# A NEW internal module added to the gateway therefore leaks NOTHING by default:
# it is simply absent from this list, so it is never synced, never packed.
#
# Format: one bundle-relative path per line (gateway/...). One glob permitted:
# the compiled license_core .so (built at publish time, name varies by ABI).
# Classification rationale per path: bundle-classification.md
# ============================================================================
